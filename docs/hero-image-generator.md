# Blog Hero Image Generator

How the app produces the hero image for every blog post: an AI-generated,
topic-relevant **isometric line-art background** (no text) with the **Flo logo
and the blog title composited on top**.

---

## 1. Overview

Each hero is a **1200×630 PNG** built in two layers:

1. **Background** — a text-free, brand-themed isometric line-art illustration
   relevant to the post's topic, generated by **Nano Banana 2**
   (`gemini-3.1-flash-image-preview`).
2. **Overlay** — the **"Flo." wordmark**, the **blog title** (the focal point),
   a `flo.finance` footer, a periwinkle accent bar, and a legibility scrim.

The two layers are flattened with **sharp**, and the final PNG is stored as
bytes in Postgres so it is shared by the web and worker containers.

If image generation is disabled or fails, the renderer falls back to a clean
cream design (no background) — generation never blocks publishing.

```
topic/title
   │
   ▼
generateHeroBackground()  ──►  Nano Banana 2  ──►  JPEG bytes (line-art, no text)
   │
   ▼
renderHeroPng()
   ├─ Satori (next/og)  ──►  transparent overlay PNG (wordmark + title + footer + bar)
   ├─ sharp             ──►  cover-fit background to 1200×630
   ├─ sharp             ──►  full-bleed legibility scrim (SVG gradient)
   └─ sharp.composite([ background, scrim, overlay ])  ──►  final PNG
   │
   ▼
Blog.heroImage (bytea in Postgres)
```

---

## 2. The model — Nano Banana 2

- **Model id:** `gemini-3.1-flash-image-preview` (Google's "Nano Banana 2").
  - Nano Banana **Pro** is `gemini-3-pro-image-preview`; the original Nano
    Banana is `gemini-2.5-flash-image`. Any of these can be selected in
    **Settings → Hero image → Image model**.
- **SDK:** `@google/genai` (`ai.models.generateContent`).
- **Image output:** requested via `responseModalities: ["IMAGE"]` and
  `imageConfig.aspectRatio: "16:9"`. The image comes back **inline** as
  base64 in `candidates[0].content.parts[].inlineData.data`.
- **Important:** the model returns a **JPEG**, not a PNG. (This matters for the
  compositing step — see §6.)

> There is **no separate `systemInstruction`** for the image model. The entire
> instruction is sent as the `contents` (the user prompt) shown in §3.

---

## 3. The prompt

The whole prompt is assembled by `buildPrompt(title, brief)` in
`lib/agent/image-gen.ts` and sent as `contents`. It interpolates the blog
**title** and, when available, the topic **brief**.

### Brand palette token

```
warm cream/ivory background (#fdf8ec), deep near-black navy ink lines (#14142b),
soft periwinkle/indigo accents (#97a6f0 and #aab4f4), with restrained use of a
single muted slate tone for depth
```

### Prompt template (verbatim)

```
Create a sophisticated editorial hero-image BACKGROUND for a fintech blog post.
The post is about: "{TITLE}".
Context: {BRIEF}.        ← line omitted when there is no brief

Visual style: elegant ISOMETRIC LINE-ART illustration — confident, clearly visible thin outlines in deep navy with periwinkle/indigo fills, forming a thematic scene that visually represents the topic (e.g. tokenized assets, on-chain finance, treasuries, vaults, infrastructure, networks — choose imagery that fits the specific topic). Clean, premium, detailed enough to read clearly.
Color theme: warm cream/ivory background (#fdf8ec), deep near-black navy ink lines (#14142b), soft periwinkle/indigo accents (#97a6f0 and #aab4f4), with restrained use of a single muted slate tone for depth.
Composition (IMPORTANT): leave the LEFT 45% of the frame as calm, almost-empty negative space — plain cream with at most a few faint thin lines (a title will be placed there). Compose the ENTIRE isometric illustration within the RIGHT 55% of the canvas, bleeding off the right edge.

STRICT REQUIREMENTS:
- Absolutely NO text, NO letters, NO numbers, NO words, NO logos, NO watermarks anywhere.
- No photorealism, no 3D render, no gradients-heavy backgrounds — flat line-art only.
- Wide 16:9 landscape banner.
```

### Why each part exists

| Instruction | Reason |
|---|---|
| "BACKGROUND … editorial hero-image" | Frames the task as a banner backdrop, not a standalone illustration. |
| Topic + brief interpolation | Makes the imagery **relevant to the specific post**. |
| "ISOMETRIC LINE-ART … thin outlines" | The requested aesthetic — sophisticated, premium, consistent. |
| Brand palette | Keeps every hero **on-brand** with the site colors. |
| "leave the LEFT 45% … empty / scene in the RIGHT 55%" | Reserves a clean zone for the title so text stays legible (see §5). |
| "NO text / letters / logos / watermarks" | The logo + title are added by us; the model must not bake in text. |
| "flat line-art only, no photorealism/3D" | Prevents heavy/busy backgrounds that would fight the text. |
| "16:9 landscape" | Matches the hero aspect ratio. |

### Example (rendered prompt)

For the post *"The Best Yield-Bearing Stablecoins: Evaluating On-Chain Cash
Alternatives"* the first three lines become:

```
Create a sophisticated editorial hero-image BACKGROUND for a fintech blog post.
The post is about: "The Best Yield-Bearing Stablecoins: Evaluating On-Chain Cash Alternatives".
Context: Comparing tokenized treasury and yield-bearing stablecoin products.
```

…followed by the identical style / palette / composition / requirements blocks.

### The generation call

```ts
const res = await ai.models.generateContent({
  model: opts.model || "gemini-3.1-flash-image-preview",
  contents: buildPrompt(opts.title, opts.brief),
  config: {
    responseModalities: ["IMAGE"],
    imageConfig: { aspectRatio: "16:9" },
  },
});

// Pull the first inline image part out of the response.
for (const part of res.candidates?.[0]?.content?.parts ?? []) {
  if (part.inlineData?.data) return Buffer.from(part.inlineData.data, "base64");
}
return null; // → caller falls back to the plain design
```

`generateHeroBackground` is wrapped in try/catch and returns `null` on any
failure, so a flaky image call never breaks blog generation or publishing.

## 4. Compositing the final hero

`renderHeroPng({ title, style, background })` in `lib/image/hero.tsx`:

1. **Overlay (Satori / `next/og` `ImageResponse`):** renders the text layer on a
   **transparent** canvas — the "Flo." wordmark (Inter 800 + periwinkle dot),
   the title (Inter 700, ≤ 560 px wide, auto-sized by length), the
   `flo.finance` footer (JetBrains Mono), and the periwinkle bottom bar.
2. **Background (sharp):** the AI JPEG is `resize(1200, 630, { fit: "cover" })`.
3. **Scrim (sharp):** a full-canvas SVG gradient (see §5) is rasterized.
4. **Flatten (sharp):** `sharp(background).composite([scrim, overlay])` → PNG.

When `background` is absent the function returns the Satori overlay directly on
a solid cream background (the fallback design).

### Fonts

Fonts are **bundled in the repo** at `assets/fonts/` (Inter 500/700/800 and
JetBrains Mono 700, `.woff`) and read from disk at render time. Bundling makes
rendering deterministic and offline — and crucially they exist in **both** the
web and worker containers (`COPY . /app`).

### Brand tokens (`lib/image/hero.tsx`)

| Token | Value | Use |
|---|---|---|
| `BG` | `#fdf8ec` | cream base / scrim color |
| `INK` | `#14142b` | wordmark + title |
| `PERIWINKLE` | `#97a6f0` | logo dot + footer |
| `BAR` | `#aab4f4` | bottom accent bar |
| `SUBTLE` | `#6e7390` | title color in the no-background fallback |

## 5. The legibility scrim

The title sits on the left, so a **left-weighted cream gradient** lightens the
left of the image while letting the art show on the right. It is rendered as a
**full-canvas SVG** and composited with sharp:

```svg
<svg xmlns="http://www.w3.org/2000/svg" width="1200" height="630">
  <rect width="1200" height="630" fill="#fdf8ec" fill-opacity="0.12"/>      <!-- light global wash -->
  <defs>
    <linearGradient id="scrim" x1="0" y1="0" x2="1" y2="0">
      <stop offset="0%"  stop-color="#fdf8ec" stop-opacity="0.96"/>
      <stop offset="36%" stop-color="#fdf8ec" stop-opacity="0.93"/>
      <stop offset="52%" stop-color="#fdf8ec" stop-opacity="0.5"/>
      <stop offset="66%" stop-color="#fdf8ec" stop-opacity="0"/>
    </linearGradient>
  </defs>
  <rect width="1200" height="630" fill="url(#scrim)"/>
</svg>
```

This pairs with the prompt's "keep the left 45% empty" instruction: the art is
already sparse on the left, and the scrim guarantees crisp text regardless of
what the model draws.

> **Why sharp and not a Satori `<div>`?** The scrim was originally a positioned
> div inside the Satori overlay, but Satori clips absolutely-positioned children
> to the **padded content box**, so the gradient stopped short of the edges and
> left a hard rectangular **seam** across the art. Rendering the scrim as a
> full-canvas layer in sharp fixed it.

---

## 6. Why sharp does the compositing

**Satori (`next/og`) cannot reliably decode raster images** — handing it the
model's image (an `<img>` with a data URI) renders **blank**, and the output is
a **JPEG** which Satori doesn't support at all. So:

- Satori is used **only** for what it is great at — laying out **text and vector
  shapes** on a transparent canvas.
- **sharp** does all raster work — decoding the JPEG, cover-fitting, rasterizing
  the scrim SVG, and flattening the layers.

`sharp` is pinned to **`0.34.5`** to match Next.js's bundled copy; otherwise two
different libvips versions load simultaneously and clash.

## 7. Orchestration & storage

`regenerateHero(blogId)` in `lib/blogs.ts` ties it together:

```ts
export async function regenerateHero(id: string): Promise<void> {
  const blog = await db.blog.findUnique({
    where: { id },
    select: { id: true, title: true, topic: { select: { brief: true } } },
  });
  if (!blog) throw new Error("Blog not found");

  const settings = await getResolvedSettings();
  const style = settings.heroStyle;

  // Generate the AI background (skipped when disabled / no API key / on failure).
  let background: Buffer | null = null;
  if (style.generateBackground !== false && settings.geminiApiKey) {
    background = await generateHeroBackground({
      apiKey: settings.geminiApiKey,
      model: style.imageModel,
      title: blog.title,
      brief: blog.topic?.brief ?? null,
    });
  }

  const png = await renderHeroPng({ title: blog.title, style, background });
  await db.blog.update({
    where: { id },
    // heroImagePath="db" is a lightweight "has hero" marker for list queries.
    data: { heroImage: new Uint8Array(png), heroImagePath: "db" },
  });
}
```

**Storage:** the final PNG lives in `Blog.heroImage` (Postgres `bytea`). This is
deliberate — the web and worker run in **separate containers with separate
filesystems**, so a local-disk file written by the worker would be invisible to
the web service. Bytes in the DB are shared and survive redeploys.

- **Served by:** `GET /api/blogs/[id]/hero` (`getHeroImage` → `image/png`).
- **Published to Ghost:** the bytes are spilled to a temp file
  (`heroTempFile`) for the Ghost Admin API image upload.

## 8. When heroes are generated

| Trigger | Path |
|---|---|
| **Automatically** after a blog is generated | worker `executeRun` → `regenerateHero` |
| **Manually** from the blog editor ("Regenerate hero") | `POST /api/blogs/[id]/hero` |
| **On publish** if a blog somehow has no hero | `publishBlog` → `regenerateHero` |
| **Backfill** existing posts | loop `POST /api/blogs/[id]/hero` per blog |

## 9. Settings

The hero is configured under **Settings → Hero image**, stored in the
`heroStyle` JSON blob (no DB migration needed):

| Field | Default | Meaning |
|---|---|---|
| `generateBackground` | `true` | Toggle the AI background on/off. |
| `imageModel` | `gemini-3.1-flash-image-preview` | Which image model to use. |
| `brand` | `flo.finance` | Footer label text. |

## 11. Design decisions / gotchas (quick reference)

- **No baked-in text in the AI image** — the model is told not to draw any text;
  the logo + title are composited by us for crisp, controllable typography.
- **Model returns JPEG** → must be handled by sharp, not Satori.
- **Satori can't decode rasters** → it only renders the transparent text layer.
- **Satori clips absolutes to the padded box** → the scrim is a full-canvas
  sharp layer to avoid a seam.
- **sharp pinned to 0.34.5** → avoids a libvips clash with Next's copy.
- **Fonts bundled** → deterministic, offline, present in both containers.
- **Hero stored in Postgres** → shared across web/worker, persists across deploys.
- **Generation is best-effort** → failures fall back to the plain design.
